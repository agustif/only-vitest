only-test

Enforce Vitest. Intercept `bun test`.

`only-test` lets a repo declare one approved test runner and, when activated, silently redirects bare `bun test` to that runner. Supports `bun-test`, `vitest`, and `jest`. 

curl -fsSL https://agustif.github.io/only-test/install.sh | bash

What it solves

`bun run test`, `npm test`, and `pnpm test` all go through project scripts. Bare `bun test` does not. `only-test` installs the missing policy boundary.

Quick start

Run `only-test install --runner vitest --sync-scripts`, then `eval "$(only-test activate)"` in the target repo.

Runners

First release supports `bun-test`, `vitest`, and `jest`. The global Bun shim silently redirects `bun test` to the configured runner.
