only-vitest Vitest-only Bun interception

Keep the rule simple: Bun calls Vitest.

`only-vitest` is the stable narrow version of this idea. It exists for teams that do not want a generic runner-policy system. The repo declares Vitest. The global Bun shim makes bare `bun test` obey that choice.

curl -fsSL https://agustif.github.io/only-vitest/install.sh | bash

One runner

No adapter registry, no runner choice screen, no generic policy surface.

Low-noise setup

Minimal repo policy, global shim by default, and optional script syncing when the team wants explicit package scripts.

Stable branch

Use this when Vitest is already your team standard and you want the smallest possible enforcement layer.

step 01

Write repo policy

Declare Vitest once so the repo has an explicit test-runner contract.

only-vitest init
step 02

Install the global shim

Set up the machine-level Bun wrapper that makes bare `bun test` predictable.

only-vitest setup-shell
step 03

Activate the path

Prepend the global only-test bin directory so Bun is intercepted in this shell.

eval "$(only-vitest activate)"
